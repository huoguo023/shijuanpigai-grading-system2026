## 🔧 技术细节

### 为什么 `pip` 命令不可用？

1. **PATH 环境变量未设置**
   - Python 安装时没有勾选 "Add Python to PATH"
   - 或者安装后 PATH 没有生效

2. **Python 安装位置**
   - 您的 Python 安装在：
     ```
     C:\Users\Administrator\AppData\Local\Python\pythoncore-3.14-64\
     ```
   - pip 在：
     ```
     C:\Users\Administrator\AppData\Local\Python\pythoncore-3.14-64\Scripts\
     ```
   - 但 Scripts 文件夹没有在 PATH 中

### 解决方案对比

| 方法 | 命令 | 是否需要 PATH |
|------|------|--------------|
| 直接调用 | `pip install xxx` | ✅ 需要 |
| Python 模块 | `python -m pip install xxx` | ❌ 不需要 |

### 为什么 `python -m pip` 可以工作？

- `python` 命令可以找到（在 PATH 中）
- `-m pip` 告诉 Python 运行 pip 模块
- 不需要 pip 命令在 PATH 中

## 💡 常见问题

### Q1: 为什么不直接修复 PATH？

**A**: 修复 PATH 比较复杂：
- 需要管理员权限
- 需要重启命令提示符
- 可能影响其他程序
- 使用 `python -m pip` 更简单

### Q2: `python -m pip` 和 `pip` 有什么区别？

**A**: 功能完全相同：
- `pip install xxx` = `python -m pip install xxx`
- `pip list` = `python -m pip list`
- `pip uninstall xxx` = `python -m pip uninstall xxx`
